# Server Edition — SELLER ONLY

Docker-based packaging for a customer who runs CLG Search on their own Linux
server, instead of the Windows desktop edition (`../README.md`). Same
source-free, licensed, machine-bound principle — see [../../SELLER-GUIDE.md](../../SELLER-GUIDE.md#server-edition-customer-runs-it-on-their-own-linux-server-not-a-windows-pc)
for the full step-by-step.

## Files

| File | Who it's for | Purpose |
|------|--------------|---------|
| `Dockerfile.api`, `Dockerfile.crawler`, `Dockerfile.web` | You (build-time only) | Multi-stage builds. A throwaway `builder` stage has the full source; only the compiled/bundled output reaches the final `runtime` stage that gets pushed. |
| `build-and-push.mjs` | You | Builds all three, verifies none leaked source, pushes to your private registry. |
| `docker-compose.server-edition.yml` | **Customer** | Pulls the pre-built images and runs them. No `build:` — never touches source. |
| `.env.example` | **Customer** | Copy to `.env`, fill in the real registry/password/ports. |
| `pg-init/` | **Customer** | First-run database schema (regenerated by `build-and-push.mjs` from `packages/database/prisma/schema.sql` — never hand-edit). |

## Quick reference

```bash
# Build + verify + push (run from the repo root, or use this relative path)
node tools/package/docker/build-and-push.mjs \
  --registry ghcr.io/your-github-username --tag v1.0.0 \
  --api-url http://<customer-server>:4100 --push
```

## Two things worth remembering

1. **`NEXT_PUBLIC_API_URL` is baked into the web image at build time** (Next.js
   requirement — public env vars are compiled into the client bundle, not read
   at runtime). If a customer's server address changes, you must rebuild and
   re-push `clg-web` with the new `--api-url`, not just edit their `.env`.
2. **License binding uses the host's `/etc/machine-id`**, mounted read-only
   into the `api` container (already wired in `docker-compose.server-edition.yml`).
   This is deliberate: a container's own hostname/MAC are reassigned by Docker
   on every recreation, so binding to those (the Windows-edition scheme) would
   silently break the license on a routine restart. See the comment in
   `packages/license/src/fingerprint.ts` (`getMachineFingerprint`) for the full reasoning.
